# Contributing

How to contribute to the ClaimSync documentation.

## Workflow

1. Create a branch from `main`.
2. Edit or add Markdown files under `docs/`.
3. Preview locally with `mkdocs serve`.
4. Open a pull request. Once merged to `main`, the site deploys automatically.

## Style

- One H1 (`#`) per page — it becomes the page title.
- Keep pages focused; split long topics into multiple pages under `docs/guides/`.
- Use relative links between docs (e.g. `[Getting Started](../getting-started.md)`).
