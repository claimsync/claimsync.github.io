# Feature Modules

`mom-backend` is organized by clinical feature module, not technical layer — see the [Architecture Guide](../CLAUDE.md) for the shared layout every module follows. This page indexes all 21 modules currently wired into `webapi/api/router.py`, grouped by what they're for.

## Core clinical scores

- **[Medication Appropriateness Index (MAI)](mai.md)** — per-medication appropriateness ratings, v1/v2 toggled by an env var
- **[Anticholinergic Burden (ACB)](acb.md)** — anticholinergic cognitive burden score
- **[Sedative Load Model (SLM)](slm.md)** — sedative burden score
- **[Fall Risk Assessment](fall-risk.md)** — fall risk scoring plus IDT care planning
- **[Opioid](opioid.md)** — daily morphine milligram equivalent (MME) and risk level
- **[Drug-Drug Interactions (DDI)](ddi.md)** — interaction matrix and medication-alternatives ranking
    - **[Medication Alternatives: Complete Reference](medication-alternatives.md)** — deep dive on the alternatives ranker: request flow, scoring, SLM 4-tier lookup, config, env vars
- **[Cascade](cascade.md)** — prescribing-cascade hypothesis detection

## Care workflows

- **[Recommendations](recommendations.md)** — maker-checker clinical recommendation workflow
- **[Medication Optimization](optimization.md)** — triage, clinical schedule checklist, medication status, PDF reports
- **[Medication Reconciliation](reconciliation.md)** — current vs. previous medication list comparison
- **[Hospitalization](hospitalization.md)** — PACE discharge review records
- **[CMR Reviews](cmr-reviews.md)** — Comprehensive Medication Review sign-off workflow

## Aggregation & reporting

- **[Hub Analysis](hub.md)** — stored risk analysis with geriatric flag enrichment
- **[Clinical Profile](clinical-profile.md)** — patient context timeline, EHR wiki, patient roster, batch stats
- **[Network Summary](network-summary.md)** — network/center-level dashboards and patient list
- **[Quality Measures](quality-measures.md)** — 17 HEDIS/quality-gap calculations
- **[Alerts](alerts.md)** — critical alert feed generated by the scheduler

## Reference & support

- **[Citations](citations.md)** — AHFS/FDA/guideline citation rendering, incl. a Weaviate-backed vector lookup
- **[Doc URL](doc-url.md)** — resolves document links via an external doc-linker service
- **[Jerimed Users](jerimed-users.md)** — Jerimed platform user accounts
- **[User Walkthrough](walkthrough.md)** — per-user product-tour completion state

!!! note
    `webapi/adherence_risk/` exists on disk but isn't documented here — its source `.py` files are gone (only compiled `.pyc` bytecode remains) and it isn't imported by `webapi/api/router.py`. It's dead/unwired, not an active module.
